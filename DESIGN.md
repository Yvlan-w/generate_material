# AI营销素材生成小程序 - 设计思考

## 气质与意象

**核心气质**：专业稳重、信任可靠、现代高效
**意象锚点**：清晨金融中心的玻璃幕墙——通透、秩序、力量感，阳光穿透时的蓝色调光晕，精密数据在屏幕上流动的节奏感。这是投资咨询行业的视觉语言：不是张扬的炫耀，而是克制的专业；不是冰冷的距离，而是理性的亲和。

## 视觉策略

**摄影/图像方向**：
- 商业场景：现代办公空间、团队协作、数据可视化图表
- 人物表现：专业着装、自信姿态、协作氛围
- 风格处理：高清质感、适度景深、光线柔和但有层次

**图形语言**：
- 卡片式布局：信息分层清晰，呼吸感充足
- 图标系统：Lucide Icons 线条风格，保持克制
- 装饰元素：最小化，仅用色块层级和微阴影区分区域

## 配色方案

| 颜色 | HSL 值 | 意象来源 |
|------|--------|----------|
| **主色 Primary** | `#1E40AF` (HSL 224, 76%, 38%) | 金融中心的蓝色玻璃幕墙，代表信任、专业、稳健 |
| **Primary Container** | `#DBEAFE` (HSL 214, 95%, 92%) | 晨光穿透玻璃的柔和蓝调，作为背景填充 |
| **Background** | `#FFFFFF` (纯白) | 干净的办公环境，最大化的内容可读性 |
| **Surface Container** | `#F3F4F6` (灰阶) | 会议室白板的辅助区域，信息层级分隔 |
| **On Surface** | `#1F2937` (深灰) | 正式文档的墨迹，确保文字清晰可读 |
| **On Surface Variant** | `#6B7280` (中灰) | 辅助信息的温和色调，不抢夺视觉焦点 |
| **Success** | `#22c55e` | 合规通过的绿色印章，行业信任信号 |
| **Warning** | `#f59e0b` | 待审核状态的警示，提醒注意 |

**色调原则**：
- 整体色调以蓝色为主轴（信任感）
- 背景与表面保持纯中性灰阶（不混入主色相）
- 通过色块层级而非边框分隔区域（现代感）
- 避免高饱和度色彩（不符合金融行业克制气质）

## 字体排版

**推荐字体**：
- `--font-sans`: Inter + Noto Sans SC + system-ui
- Inter：现代、清晰、专业（拉丁字符）
- Noto Sans SC：中文场景的优雅对应
- system-ui：兜底保证跨端可用

**排版节奏**：
- 标题（L1）：`text-xl font-bold` - 页面级大标题，20px
- 卡片标题（L2）：`text-base font-semibold` - 功能区块标题，16px
- 正文（L3）：`text-sm font-normal` - 描述文字、表单标签，14px
- 辅助信息（L4）：`text-xs font-normal` - 时间戳、状态标签，12px

**行高与间距**：
- 标题行高 1.25（紧凑有力）
- 正文行高 1.5（阅读舒适）
- 层级间间距跳变明显（gap-6~8）

## 动效与交互

**运动性格**：
- 过渡曲线：ease-out（快速启动、平滑结束）
- 悬停反馈：`transition-colors` 150ms
- 点击反馈：`active:` 状态即时响应

**交互节奏**：
- 按钮 hover：背景色平滑过渡
- TabBar 切换：选中态颜色 + 字重变化
- 输入框 focus：ring-2 ring-primary/30 光晕反馈

**禁忌**：
- 禁止弹性动画（bounce）- 不符合专业气质
- 禁止旋转/闪烁动画 - 会干扰用户决策
- 禁止页面级淡入淡出 - 会拖慢交互节奏

## 设计禁忌

**视觉禁忌**：
- ❌ 禁止高饱和度渐变背景（如紫蓝渐变）
- ❌ 禁止圆形装饰元素、半透明弧形装饰
- ❌ 禁止 emoji 作为系统图标
- ❌ 禁止实线高对比边框（必须弱化为 Ghost Border）
- ❌ 禁止透明 input 背景（必须 bg-surface-container）

**布局禁忌**：
- ❌ 禁止全局等距 spacing（必须疏密有致）
- ❌ 禁止在 Header 与内容区之间添加 border-b 分隔线
- ❌ 禁止 Header 使用 fixed 定位（必须 sticky top-0）
- ❌ 禁止在 TabBar 中混入操作型按钮

**内容禁忌**：
- ❌ 禁止 Lorem ipsum 占位内容
- ❌ 禁止页面默认展示空状态
- ❌ 禁止通过 JS render 函数动态渲染列表（必须硬编码为 HTML）

## 组件选型原则

**优先使用 `@/components/ui/*`**：
- Button：所有操作按钮（提交、确认、取消）
- Card：信息卡片、列表项容器
- Input/Textarea：用户输入控件
- Slider：参数调整滑块
- Tabs：分段切换、内容分类
- Dialog：弹窗、确认框
- Toast：操作反馈提示
- Badge：状态标签（合规、待审核）
- Separator：区域分隔线

**禁止手搓通用组件**：
- 不使用 View/Text + Tailwind 手搓按钮外观
- 不在页面内临时造轮子（如自定义滑块、弹窗）

## 小程序约束

**包体积优化**：
- 图片资源必须上传到 TOS 对象存储
- 仅 TabBar 图标允许本地 PNG（微信强制要求）
- 禁止将大图/视频打包到项目

**性能要点**：
- 使用 `sticky top-0` 而非 `fixed`（减少重绘）
- 卡片阴影克制使用（shadow-card 仅在必要场景）
- 优先用色块层级而非 shadow 分隔区域

**合规要求（投资咨询行业）**：
- 所有生成图片必须经过合规审核
- 禁止包含：收益承诺、夸大宣传、敏感数据
- 状态标签：合规通过（绿色）、待审核（黄色）
